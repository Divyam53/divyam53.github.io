The $360K Cache Cost Migration: Redis to Dragonfly
DragonflyDB Redis Infra Cost Control Zero-Downtime Migration

Background: Meesho's ranking and exploration engine caches large amounts of vector-like product feeds. As traffic grew, the Redis caching layer became a massive bottleneck and driving prohibitively high memory footprint and infrastructure costs.

Technical Strategy: Led the evaluation, profiling, and production migration from Redis to DragonflyDB, a multi-threaded Redis-compatible in-memory store. Evaluated thread synchronization, pipeline performance, and latency under peak load.

Data Parity Validation: Implemented a zero-downtime migration protocol utilizing a shadow-write setup. Dual-wrote all cache transactions, validated data parity asynchronously, and shifted read traffic only after ensuring perfect key-value alignment.

Metrics & Impact
  • 3M+ IOPS caching throughput
  • 90% reduction in cache nodes
  • $360,000 saved annually
